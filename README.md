# 🍳 "Что приготовить?" — бот для жены

Двуязычный (RU / UZ) телеграм-бот-помощник по готовке: что приготовить из
того, что есть дома (текстом или по фото продуктов), быстрые рецепты,
меню на неделю, список покупок и любимые блюда семьи. Учитывает узбекский
базар и местные продукты.

Мозг бота — Google Gemini API (gemini-2.5-flash), а не Claude. Причина
простая: у Google есть по-настоящему бесплатный тариф с поддержкой фото,
а у Claude API бесплатного тарифа нет (там платят за каждый запрос). Жена
может присылать фото содержимого холодильника или список продуктов
текстом — работает одинаково в любом сценарии меню.

### О бесплатности честно

- Бесплатный тариф Gemini (на июль 2026): у модели gemini-2.5-flash лимит
  примерно 10 запросов в минуту и 1500 в сутки — для одной семьи это с
  большим запасом, кончиться не должно.
- Ключ получаешь на https://aistudio.google.com/apikey обычным Google-
  аккаунтом, банковская карта не нужна.
- Нюанс: на бесплатном тарифе Google оставляет за собой право использовать
  присланные фото и тексты для улучшения своих моделей (это прописано в их
  условиях использования). На платном тарифе такого нет. Для фото
  содержимого холодильника это обычно не критично, но лучше знать заранее.
- Хостинг на Railway — отдельная история, к ИИ не относится: у тебя уже
  есть аккаунт под бота для Lublin, но сам Railway не бесплатен бесконечно
  (после пробного лимита — по факту использования). Если нужен полностью
  бесплатный хостинг — можно рассмотреть альтернативы (бесплатный VPS-тир
  некоторых провайдеров, или запуск на своём компьютере/Raspberry Pi через
  polling) — скажи, если интересно, распишу варианты.

## Что внутри

```
bot.py         — обработчики Telegram (меню, кнопки, диалоги)
texts.py       — все тексты на русском и узбекском (латиница)
storage.py     — SQLite: пользователи, любимые блюда, список покупок, история
ai_client.py   — обращение к Claude API за рецептами
requirements.txt
Procfile       — команда запуска для Railway
.env.example   — какие переменные окружения нужны
```

## 1. Получить токены

**Telegram-бот:**
1. Напиши @BotFather → `/newbot` → следуй инструкциям → получишь токен вида `123456:AAAA...`

**Gemini API ключ (бесплатно):**
1. Зайди на https://aistudio.google.com/apikey
2. Войди обычным Google-аккаунтом → Create API Key
3. Готово — банковская карта не нужна, лимитов с головой хватит для семьи

## 2. Запуск локально (проверить перед деплоем)

```bash
cd chto-prigotovit-bot
pip install -r requirements.txt
cp .env.example .env
# открой .env и впиши TELEGRAM_BOT_TOKEN и GEMINI_API_KEY
python bot.py
```

Напиши боту `/start` в Telegram — должен появиться выбор языка и главное меню.

## 3. Деплой на Railway

1. Залей папку в GitHub-репозиторий (как делал с ботом для Lublin).
2. В Railway: New Project → Deploy from GitHub repo → выбери репозиторий.
3. В Variables добавь:
   - `TELEGRAM_BOT_TOKEN`
   - `GEMINI_API_KEY`
   - `GEMINI_MODEL` (необязательно, по умолчанию `gemini-2.5-flash`)
   - `DB_PATH` = `/data/bot.db`
4. **Важно про базу данных:** файловая система Railway сбрасывается при
   каждом редеплое. Чтобы любимые блюда, язык и список покупок не
   терялись — подключи **Railway Volume**:
   - Settings → Volumes → Add Volume → Mount path: `/data`
   - Убедись, что `DB_PATH=/data/bot.db` (уже указано выше)
5. Railway должен сам подхватить `Procfile` и запустить `python bot.py`
   как worker-процесс (не web — боту не нужен открытый порт).
6. Deploy. В логах должно появиться `Бот запущен`.

## 4. Как этим пользоваться (для жены)

- `/start` — выбрать язык (потом можно сменить в Настройках)
- Главное меню — 12 кнопок под все сценарии: что есть дома, по времени,
  по продукту, "удиви меня", чего не хватает, на сколько человек, по
  настроению, по бюджету, меню на неделю, любимые блюда, список покупок,
  настройки
- В сценариях "Что есть дома" и "Не хватает ингредиентов" можно как
  написать список продуктов текстом, так и просто прислать фото — бот сам
  разберёт, что на фото, и даст рецепты. Фото можно прислать даже без
  нажатия кнопки — бот всё равно поймёт, что это список продуктов
- В Настройках стоит один раз указать размер семьи по умолчанию,
  аллергии и нелюбимые продукты — бот будет их учитывать во всех рецептах
- В "Любимые блюда" можно добавить, кто что любит (например `муж — донер`) —
  бот со временем начнёт предлагать это чаще и не повторяться

## 5. Возможные доработки (когда будет время)

- Кнопка "🔁 Другой вариант" под рецептом, если что-то не понравилось
- Напоминание раз в неделю "Составить меню на неделю?" (через JobQueue)
- Экспорт списка покупок в текст одним сообщением для копирования
